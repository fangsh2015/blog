  1. dockerfile中，CMD和 ENTRPPOINT 的区别
    可以通过 entrypoint 来执行命令 cmd为前面的命令输入参数

cmd
设置容器启动后默认执行的命令和参数 定义多个cmd，则执行最后一个。docker run 启动一个容器时指定了执行的命令，则Dockerfile文件中的CMD命令将会被忽略。
entrypoint 设置容器启动时运行的命令；让容器以应用程序或者服务的形式运行。entrypoint不会被忽略，全部会执行。

run
执行命令并创建新的Image Layer

RUN apg-get install -y vim
CMD echo “hello docker”
ENTRYPOINT echo “hello docker”

FROM centos
ENV name DOCKER
ENTRYPOINT echo “hello $name”

FROM centos
EVN name DOCKER
CMD echo “hello $name”

sudo docker run -d –name test1 busybox /bin/sh -c “while true ; do sleep 3000; done”

/bin/sh 相当于 /bin/bash –posix
positx特定规范为：当某行代码出错时，不继续往下执行

启动容器时 -e为容器设置环境变量
dockr run -e java_home=xxxxx –name test_java -d java
